Project Spotlight

ClimateTrack — 커뮤니티 기반 자연재해 경보 플랫폼

문제 정의 (Problem)

기존의 자연재해 관련 정보는 뉴스, SNS, 정부 기관 사이트 등 여러 플랫폼에 분산되어 있어, 사용자가 재난 발생 시 즉각적으로 필요한 정보를 얻기 어렵다는 문제가 있었습니다.

이로 인해 대응이 늦어지고, 재난 상황에 대한 지역 사회의 인식과 대응 효율성이 떨어지는 한계가 있었습니다. ClimateTrack은 이러한 문제를 해결하기 위해, 분산된 재난 정보를 통합·시각화하고 사용자 위치 기반으로 맞춤형 알림을 제공하는 커뮤니티 중심의 통합 재난 대응 시스템으로 기획되었습니다.

아키텍처/설계 (Architecture)

ClimateTrack의 시스템은 데이터 수집 → 처리 및 저장 → 알림 → 시각화의 4단계 구조로 설계되었습니다. 외부 API로부터 실시간 기상·재난 데이터를 수집하고, Spring Boot 기반 백엔드 서버가 이를 처리하여 MongoDB에 저장합니다. 백엔드에서는 사용자 위치와 재난 발생 지점을 비교하여, 지정 반경(예: 10km 이내)에 해당할 경우 푸시 알림을 자동으로 전송하는 로직을 구현했습니다. 수집된 데이터는 React Native 기반 모바일 앱과 웹 대시보드에서 지도 및 커뮤니티 기능으로 시각화되어 제공됩니다. 전체 시스템은 자동화된 데이터 흐름을 통해 실시간 재난 감지 및 대응이 가능하며, 개인 사용자뿐 아니라 기관에서도 활용 가능한 지능형 재난 정보 통합 플랫폼으로 설계되었습니다.

핵심 기능 (Core Features)

실시간 재난 데이터 수집 및 통합

외부 기상청 및 재난 관련 API를 주기적으로 호출하여 최신 데이터를 수집하고, Spring Boot 백엔드 서버에서 데이터 파싱 및 구조화 과정을 거쳐 MongoDB에 저장했습니다. Sheduler를 활용한 자동화된 데이터 갱신 구조를 설계하여, 사용자는 언제나 최신 재난 정보를 확인할 수 있습니다.

사용자 위치 기반 푸시 알림 시스템

사용자의 위치 정보와 재난 발생 지점을 비교하여, 지정된 반경(예: 10km 이내)에 포함될 경우 Firebase Cloud Messaging (FCM)을 통해 푸시 알림을 전송하도록 구현했습니다. 이 기능을 통해 사용자는 재난 발생 시 즉각적인 알림을 받을 수 있으며, 백엔드에서 조건을 자동 판단하여 실시간 대응 시스템을 구축했습니다.

커뮤니티 및 시각화 대시보드

모바일 앱과 웹 대시보드를 통해 사용자가 재난 상황을 지도 기반으로 시각적으로 확인할 수 있도록 설계했습니다. 또한 지역 커뮤니티 기능을 도입하여, 사용자가 직접 재난 상황을 공유하거나 의견을 나눌 수 있게 하여 참여형 정보 생태계를 구축했습니다.

나의 역할 및 기여

  • 사용자 간 소통이 가능한 커뮤니티 페이지 기능 구현
  • 사용자 위치를 기반으로 주변 게시글을 표시하는 근접 커뮤니티 피드 페이지 개발
  • 앱의 메인 UI/UX 디자인 설계, 특히 홈 화면에서 사용자 제보 및 공식 보도 내용을 색상 마커로 표시하는 인터랙티브 지도 화면 구현
  • 아이콘 디자인 및 시각적 아이덴티티 구성
  • 위 기능들에 필요한 백엔드 API 및 데이터 처리 로직 구현 (Spring Boot 기반)

결과 및 성과

  • 실시간 데이터 자동 업데이트 주기 1분 단위로 안정화 가능
  • 사용자 위치 기반 알림 정확도 90% 이상 달성
  • 심플하고 직관적인 UI 설계로 사용자가 쉽게 접근하고 이용할 수 있는 인터페이스 구현
  • 시연 평가에서 “데이터 구조와 알림 시스템의 완성도” 및 “UI의 명확성과 사용 편의성”에 대해 긍정적 피드백 수령

회고 (배운 점)

  • 실시간 데이터 처리와 비동기 통신 구조의 중요성을 이해하고, Spring Boot의 스케줄링과 REST 통신 구조를 깊이 있게 다룰 수 있었습니다.
  • 프런트엔드와 백엔드 간의 데이터 흐름을 실시간 API 기반으로 연결하며, 실제 환경에서 데이터가 어떻게 교환되고 처리되는지에 대해 많은 경험을 쌓았습니다.
  • 사용자 중심의 UI 설계에 대해 더 깊이 배우며, 디자인의 심미성과 사용성(Usability)의 균형을 유지하는 방법을 익혔습니다.
  • 커뮤니티 중심의 앱을 개발하면서, 사용자 참여를 유도하고 공동체의 상호작용을 촉진하는 설계 방식의 중요성을 깨달았습니다.
